# Knowledge Capture Skill

Whenever a user asks a technical question:

1. Answer the question normally.

2. Then maintain a notes.md file in the nearest relevant directory.

3. Before writing:

   * Search for existing notes.md files.
   * Check whether the topic already exists.
   * Merge instead of duplicating.

4. Notes must not contain:

   * Raw conversations
   * Q&A transcripts
   * Temporary debugging details

5. Notes must contain:

   ## Key Insight

   Core understanding.

   ## Mental Model

   Intuitive explanation.

   ## Example

   Minimal working example. Keep code minimal.

   ## Common Mistakes

   Typical pitfalls.

   ## Revision

   3-5 bullets for rapid review.

6. Optimize notes for:

   * Future retrieval
   * Fast revision
   * Long-term learning

7. Prefer improving existing notes over creating new files.

8. Keep notes concise and evergreen.

9. Write notes as learning records, not mistake logs. Focus on what was understood, with only small mentions of pitfalls when useful.
